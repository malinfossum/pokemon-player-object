# Pokémon Player Object

A simple JavaScript MVC project that demonstrates how related player data can be grouped into an object and rendered in the UI.

---

## Run Locally

Open `index.html` directly in your browser
or use Live Server in VS Code.

---

## Features

- Player data stored in a single object
- Current Pokémon displayed in the UI
- Caught Pokémon stored in an array
- Caught Pokémon list rendered dynamically
- Basic MVC-inspired structure

---

## Documentation

This project demonstrates how to group related data into an object instead of using multiple standalone variables.

It also practices working with arrays, loops, and dynamic rendering in the UI using a simple MVC-inspired structure.

The main architectural decision was to store player-related data inside one `player` object, making the code easier to read, maintain, and extend.
